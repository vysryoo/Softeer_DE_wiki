
# [2026-07-16] 회고

## Review (리뷰)
 - 날짜와 시간은 목적에 맞게 DATE와 DATETIME을 구분하고, 참/거짓 값은 추후 상태 확장을 고려해 코드성 컬럼으로 설계하는 것이 안전함을 알게 되었다.
 - 동일한 개념이라도 DBMS마다 지원 타입과 세부 동작이 다르므로, 도메인 기준을 문서화하고 매뉴얼을 반드시 확인하는 습관의 중요성을 배웠다.

## 잘한 점

 - 당장의 구현뿐만 아니라, 추후 데이터 확장(상태값 추가 등)이나 마이그레이션까지 고려하여 설계해야 한다는 데이터베이스 설계의 기본 원칙을 세웠다.


## 아쉬운 점
 - 글로벌 서비스에서 문제가 될 수 있는 TIMESTAMP의 타임존(Timezone) 처리 방식에 대해서는 실제 프로젝트에 적용해 보면서 추가적인 심화 학습이 필요할 것 같다.
 - 배운 개념을 바탕으로 기존에 진행했던 미니 프로젝트나 실습 코드의 DB 스키마를 직접 리뷰하고 수정해 보는 시간을 아직 가지지 못했다.
